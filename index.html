Project Overview

Voxio acts as a serverless, decentralized network node that establishes an ad-hoc encrypted mesh using a hybrid protocol architecture. It utilizes UDP Broadcasting for "Zero-Config" peer discovery, allowing nodes to automatically handshake on a LAN without manual IP entry. Once discovered, peers form stateful TCP Connections to create a resilient daisy-chain topology, ensuring reliable delivery where every user acts as both a client and a router.

The system implements a robust "Gossip" protocol driven by a Controlled Flooding algorithm. To prevent network congestion and infinite loops (the Count-to-Infinity problem), all packets utilize a Time-To-Live (TTL) mechanism. A key feature of Voxio is its built-in Application-Layer Traceroute, a diagnostic tool that parses packet metadata to reconstruct and visualize the exact multi-hop path a message takes through the mesh, verifying active routing logic in real-time.

Security and user experience are prioritized through modern cryptographic standards and a rich terminal interface. All private communications are secured using the Sodium library's high-speed elliptic curve cryptography (Curve25519), guaranteeing end-to-end privacy. The architecture is further hardened with internal traffic policing and DoS Protection, enforcing strict 4KB buffer limits to prevent memory bombs. Powered by non-blocking, multi-threaded I/O, Voxio ensures the interface remains responsive even under heavy network traffic.

Key Features

Ad-Hoc Mesh Topology

Implements a "Gossip" protocol where every peer acts as a router. If a direct link fails, the mesh automatically reroutes messages through neighbors via a daisy-chain.

Poly1305 Integrity

Beyond standard encryption, every packet is signed with a Poly1305 Message Authentication Code (MAC) to prevent tampering during relay hops.

Zero-Config Discovery

Utilizes UDP Broadcasting to instantly detect local peers. The system handles automatic handshakes, allowing users to join the mesh without manual configuration.

Overlay Traceroute

A built-in network diagnostic tool that appends node signatures to packet headers, allowing the receiver to visualize the exact path (Trace) the message took.

DoS Traffic Policing

Features internal buffer limits (4KB) and strict disconnect logic to protect the node from memory bombs and malformed packets.

TTL & Flooding

Uses a Controlled Flooding algorithm with Time-To-Live (TTL) counters to propagate messages across segmented networks while preventing routing loops.
